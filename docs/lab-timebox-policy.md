# Lab Timebox Policy

## Purpose

Every PythonLabs lab should be designed to fit within a realistic learning session.

The target duration is **60 to 90 minutes maximum**.

The lab should be practical and focused. It should not become a multi-day project unless it is explicitly split into separate labs.

## Core Rule

Each lab should be scoped for:

```text
Minimum target: 60 minutes
Maximum target: 90 minutes
```

If a lab cannot reasonably fit into 90 minutes, it must be reduced, split, or moved into a later drilling/capstone lab.

## What Must Fit Inside the Timebox

The 60–90 minute timebox includes:

* reading the scenario and requirements
* checking reference material at a high level
* completing Part 1 — New Chapter Content
* completing Part 2 — Cumulative Repetition
* performing the required break/fix work
* running tests or manual checks
* capturing evidence
* answering the final reflection questions

## Scope Control

To keep labs inside the timebox:

* each lab should have one primary new concept
* Part 2 should reuse older topics without becoming a second full project
* break/fix should be meaningful but controlled
* tests should be focused and relevant
* optional stretch tasks should be clearly marked as optional
* long projects should be split across multiple labs

## Expected Time Budget

Use this rough budget when designing labs:

| Section | Target time |
| --- | --- |
| Read scenario, requirements, and references | 10–15 minutes |
| Part 1 — New Chapter Content | 20–25 minutes |
| Part 2 — Cumulative Repetition | 15–20 minutes |
| Break/Fix | 10–15 minutes |
| Testing and verification | 10 minutes |
| Reflection and evidence handoff | 5–10 minutes |

The exact timing can vary, but the total should remain within 60–90 minutes.

## Optional Stretch Work

Some labs may include stretch tasks, but stretch work must not be required for completion.

Use this wording:

```text
Optional stretch task — only attempt this if the required lab is complete and still within your available time.
```

Stretch tasks should not block lab completion.

## Capstone Exception

Lab 30 is the final SRE capstone and may be larger than a normal lab.

However, it should still be designed as a controlled capstone session, not an unlimited project.

If Lab 30 becomes too large, it should be broken into clearly marked phases, with each phase having its own completion criteria.

## Completion Standard

A lab is correctly scoped only if a learner can reasonably complete the required work in 60–90 minutes while still understanding the story, breaking and fixing something, testing the result, and explaining what happened.

## Final Rule

Depth matters, but scope control matters too.

Every lab should be useful, story-driven, tested, break/fix oriented, and still small enough to finish in one focused session.
